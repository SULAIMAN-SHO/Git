أولاً: تهيئة المشروع كمستودع جديد git init

ثانياً: عرض حالة الملفات (المتابعة وغير المتابعة) git status

ثالثا: إضافة ملف محدد (مثلاً index.html) للمتابعة git add index.html

رابعاً: إيقاف متابعة ملف معيّن بدون حذفه من الجهاز git rm --cached style.css

خامساً: إضافة جميع الملفات الموجودة في المشروع git add .

سادساً: حفظ التغييرات مع رسالة توضيحية git commit -m "أول حفظ للتغييرات"

إعداد اسم المستخدم العالمي في Git git config --global user.name "SULAIMAN"

إعداد البريد الإلكتروني العالمي في Git git config --global user.email "sulaiman.lutfi.sho@gmail.com"

عرض اسم المستخدم الحالي في إعدادات Git git config user.name

عرض البريد الإلكتروني الحالي في إعدادات Git git config user.email

حفظ التغييرات مع رسالة توضيحية git commit -m "add text"

عرض سجل جميع عمليات الحفظ (commits) git log

عرض سجل الحفظ (commits) بشكل مختصر وسطر واحد لكل عملية git log --oneline

الانتقال بين الفروع أو استرجاع الملفات من نسخة سابقة
git checkout

عرض جميع الفروع في المشروع (المحلية والبعيدة)
git branch -a

إنشاء فرع جديد باسم "student"
git branch student

إعادة تسمية الفرع الحالي إلى اسم جديد
git branch -m sulaiman-sho

حذف فرع باسم "student" إذا كان مدمج بالكامل
git branch -d student

حذف فرع باسم "student" بالقوة حتى لو غير مدمج ونركز على ان حرف ال D كبير وليس صغير
git branch -D student

إنشاء فرع جديد باسم "new_branch_test" والانتقال إليه مباشرة
git checkout -b new_branch_test

إعادة تسمية الفرع الحالي إلى "main" بالقوة حتى لو كان الاسم موجود مسبقًا
فعلنا ذلك عن طريق وضع حرف الـ M كبير
git branch -M main

إضافة مستودع بعيد (remote) وربطه بالمشروع المحلي باسم "origin"
git remote add origin https://github.com/SULAIMAN-SHO/Git.git
فعلنا ذلك عن طريق تحديد الرابط الخاص بالمستودع على GitHub وربطه باسم "origin"

رفع الفرع "main" إلى المستودع البعيد "origin" وربطه كفرع افتراضي git push -u origin main
فعلنا ذلك عن طريق استخدام الخيار -u ليتم تعيين "main" كفرع افتراضي للعمليات القادمة

رفع الفرع "main" إلى المستودع البعيد "origin" وربطه كفرع افتراضي للعمليات القادمة
git push -u origin main
فعلنا ذلك عن طريق استخدام الخيار -u ليتم تعيين "main" كفرع افتراضي بحيث في المستقبل يكفي كتابة git push أو git pull بدون تحديد اسم الفرع
